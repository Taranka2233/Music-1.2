# N54 // AUDIO DECK

Локальный аудиоплеер в эстетике Cyberpunk 2077. Матовый, круглый, без единого неона.
Один HTML-файл, ноль зависимостей кроме `jsmediatags`. Треки не покидают устройство.

Работает двумя способами:

| | Браузер / Pages | APK (Capacitor) |
|---|---|---|
| Откуда музыка | ты выбираешь папку, дека проходит её насквозь | **MediaStore — вся музыка телефона разом** |
| Где живут файлы | blob в IndexedDB | не копируются, играются по `content://` |
| Экран блокировки | Media Session (в Chrome) | нужен плагин, см. ниже |
| 40 ГБ библиотеки | не влезет в квоту | пара сотен КБ метаданных |

## Что внутри

**Шторка и экран блокировки.** Обложка, название, кнопки — в шторке и на локскрине,
плюс управление с кнопок наушников. Android WebView не умеет Media Session Web API,
поэтому в APK работает нативный плагин, а в браузере — сам веб-стандарт. Дека выбирает
путь сама. Плагин же поднимает foreground service: без него Android усыпляет вебвью
через пару минут после блокировки экрана.

**Плеер целиком.** Библиотека (треки / альбомы / артисты / избранное), поиск, 6 сортировок,
плейлисты, очередь с перестановкой, shuffle, repeat, петля A–B, скорость 0.5–2×,
кроссфейд 0–12 с, ID3-теги с обложками и текстами, громкость.

**Радио Найт-Сити.** Шесть станций, которые собираются **из твоей же библиотеки**
по полю `genre` в тегах. Переключение через настоящий белый шум: буфер случайных
сэмплов + bandpass, свипующий с 500 Гц до 4.8 кГц. Станция без совпадений честно
пишет «ПОМЕХИ», а не притворяется.

**Кибервер.** Эквалайзер как слоты имплантов, 8 пресетов. АЧХ посчитана по формулам
Audio EQ Cookbook — отклонение 1.6–2.3 дБ от цели, это наложение октавных полос, норма.
Q подобран перебором: 1.2. Плюс компрессор и расширитель стерео (эффект Хааса).

**Street Cred.** Слушаешь → капает XP → НОМАД → УЛИЧНЫЙ ПЁС → СОЛО → ФИКСЕР →
НЕТРАННЕР → ЛЕГЕНДА НАЙТ-СИТИ. Каждый ранг открывает облик корпорации: вся палитра
перекрашивается (Arasaka, Militech, Kang Tao, NetWatch, Night Corp).

**Kiroshi.** Долгое нажатие на треке — сканлайн проходит по карточке и вываливает
техничку: битрейт, формат, размер, год, счётчик прослушиваний.

**Breach Protocol.** При старте деки взламывается буфер: матрица гексов, путь
подсветки, шкала.

**Вертушка.** Обложка круглая, поэтому крутится как пластинка. Темп следует за
скоростью воспроизведения. На паузе не встаёт колом, а выбегает за 3 секунды —
инерция 0.035, как у настоящей. Кольцо прогресса и фон дышат на низах: первые бины
с анализатора → CSS-переменная. Не таймер и не рандом — форма волны, которая
прямо сейчас идёт через мастер-выход.

**Flatline.** Таймер сна: за 20 секунд звук уходит в ноль, дека «умирает».

Всё уважает `prefers-reduced-motion`.

## Запуск

### Веб

```bash
npm run serve          # http://localhost:5173
```

Или просто открыть `www/index.html`. GitHub Pages поднимается сам из `.github/workflows/pages.yml`
после включения Pages в настройках репозитория (Source → GitHub Actions).

### APK

```bash
npm install
npx cap add android                     # папки android/ в репозитории нет
npx @capacitor/assets generate --android # иконки и сплэш из assets/
node scripts/patch-android.mjs          # плагин + разрешения
npx cap sync android
cd android && ./gradlew assembleDebug
```

Иконка собирается из `assets/icon.png` (обычная), `assets/icon-foreground.png` +
`assets/icon-background.png` (адаптивная, знак ужат в безопасную зону 66% — Android
режет края под форму лаунчера) и `assets/splash.png`.

APK ляжет в `android/app/build/outputs/apk/debug/app-debug.apk`.

Либо просто запушить в `main` — workflow `APK` соберёт и приложит артефакт.

## Как устроен натив

`android/` **не хранится в git** — её создаёт `cap add android` на каждой сборке.
Всё нативное живёт в `native/` и `scripts/patch-android.mjs`, который идемпотентно
накатывает изменения поверх сгенерированного проекта:

1. `native/MusicScannerPlugin.java` → `android/app/src/main/java/<pkg>/`
2. `MainActivity.java` перезаписывается с `registerPlugin(MusicScannerPlugin.class)`
3. В `AndroidManifest.xml` добавляются разрешения

### Про сканирование

Плагин **не обходит файловую систему**. Android уже обошёл её: системный MediaScanner
индексирует всё аудио в базу MediaStore. Плагин делает SQL-запрос к готовому индексу —
это быстрее и это единственный путь, который пропускает Google Play.

Разрешения: `READ_MEDIA_AUDIO` (Android 13+) и `READ_EXTERNAL_STORAGE` с
`maxSdkVersion=32` для старых. Два разных алиаса в плагине намеренно: просить оба
разом нельзя, на API 33+ старое молча не выдаётся и запрос виснет.

**`MANAGE_EXTERNAL_STORAGE` не используется.** Он дал бы сплошной обход диска, но Google
Play разрешает его файловым менеджерам и антивирусам. Плеер с ним отклонят.

Связка с вебом — через `Capacitor.convertFileSrc()`: `content://` переписывается в
`http://localhost/_capacitor_content_/…`, который WebView скармливает в `<audio src>`.

## Известные ограничения

**Лицензия плагина media-session — GPL-3.0, а репозиторий MIT.** Java-код плагина
компилируется в APK. GPL-3 требует, чтобы вся распространяемая работа была под GPL-3,
поэтому ярлык MIT на этом репозитории формально неточен. Варианты: сменить лицензию
проекта на GPL-3.0, либо взять [`@capgo/capacitor-media-session`](https://github.com/Cap-go/capacitor-media-session)
— поддерживаемый форк под MPL-2.0 (файловый копилефт, с MIT уживается), API тот же,
менять только строку в `package.json`. Версия у Capgo следует за мажором Capacitor,
то есть для Capacitor 6 нужна ветка 6.x — её существование не проверено.

**Фоновое воспроизведение может отваливаться минуты через три.** Открытый баг
[jofr#11](https://github.com/jofr/capacitor-media-session/issues/11): на части устройств
Android усыпляет процесс, несмотря на foreground service. `WAKE_LOCK` в манифест
добавлен, гарантий нет. Плагин не обновлялся около года.

**Эфир идёт мимо Кибервера.** Чужой поток через `createMediaElementSource` отдаёт нули,
если сервер не прислал CORS — так написано в спеке Web Audio, и иконы Icecast эти
заголовки не шлют. Поэтому радио играет через отдельную деку `#a2`, не подключённую
к аудиографу. Цена: на эфире не работают эквалайзер, компрессор, расширение стерео
и визуализатор. Название трека из потока браузеру тоже недоступно (ICY-метаданные) —
показывается имя станции.

**Обложки из сети покрывают не всё.** MusicBrainz — открытая база, но русский рэп
в ней представлен плохо. Часть треков останется без обложки.

**Рантайм Capacitor обязан лежать в `www/capacitor.js`.** Android впрыскивает в WebView
только `native-bridge.js`, а `registerPlugin` и `isNativePlatform` живут в пакете
`@capacitor/core`. Без сборщика они в страницу не попадают, и весь натив молча умирает:
`NATIVE` становится `false`, дека выглядит как обычная веб-версия. Файл кладёт сборка
(шаг «Рантайм Capacitor в www/»), в git его нет. `patch-android.mjs` проверяет наличие
и валит сборку, если файла нет. Внутри APK дека дополнительно кричит тостом, если мост
не поднялся.

**MediaStore обновляется не мгновенно.** Только что скинутый по USB трек может не
появиться до перезагрузки. Лечится `MediaScannerConnection.scanFile()`.

**Форматы.** Браузер тянет MP3, M4A, FLAC, OGG, OPUS, WAV. Мимо: WMA, APE, MIDI, ALAC.
Сканер проверяет через `canPlayType` до импорта, пропускает и говорит об этом в отчёте.

**Полоса 32 Гц в эквалайзере ничего не делает на телефоне.** Полка на +7 дБ даёт
0.57 дБ на 60 Гц. Это физика, не баг: ниже 40 Гц телефон и большинство наушников
просто не играют. Весь бас в пресетах тянет полоса 64.
